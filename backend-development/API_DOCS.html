ERP API Documentation

Base URL default: http://localhost:3000

Semua endpoint JSON. Gunakan header Content-Type: application/json untuk request dengan body.

Autentikasi

  • GET /auth/login → Redirect ke Microsoft login.
  • GET /me → Cek session saat ini. Return roles yang aktif.

Catatan

Untuk menguji endpoint terproteksi di Postman, login lewat browser ke /auth/login → setelah sukses, salin cookie erp.sid ke Postman.

Trigger DB otomatis memperbarui status termin (UNPAID/PARTIAL/PAID) dan status invoice (UNPAID/PARTIAL/PAID).

Panduan Pengujian di Postman

  1. Login via browser ke /auth/login, lalu cek /me. Salin cookie erp.sid ke Postman.
  2. Buat lead via POST /api/v1/leads/public.
  3. List leads via GET /api/v1/leads (perlu role sesuai).
  4. Claim lead via POST /api/v1/leads/:id/claim (perlu role).
  5. Buat deal via POST /api/v1/deals + tambah items jika perlu.
  6. Approve deal via PUT /api/v1/deals/:id body {"approvedBy": true}.
  7. Buat invoice via POST /api/v1/invoices + installments.
  8. Catat pembayaran via POST /api/v1/payments (isi installment_id untuk update termin).
  9. Cek status invoice dan progress via GET /api/v1/invoices/:id.

Catatan

  • Trigger di leads mengunci klaim (tidak bisa diubah setelah terisi) dan auto set claimed_at.
  • Trigger di deals mengisi approved_at saat status berubah ke APPROVED.
  • Trigger di payments menyinkronkan status termin dan invoice.
